Chapter 3: Functions.

Defining a Function

A fuction definition is a regular binding where the value of the vinding is a function. A function is created with an expression that starts with the keyword function. Functions have a set of parameters and a body. The function body of a function created this way must always be wrapped in braces, even when it consists of only a single statement. A function can have multiple parameters or no parameters at all. 

const square = function(x) {
      return x * x;
    };

Some functions produce a value, and other's produce an effect. A return keyword without an expression affter it will cause the funciton to return undefined. Parameters to a function behave like regular bindings, but their initial values are given by the caller of the funcitons, not the code in the function itself.

Bindings and Scopes

Each binding has a scope. Bindings created for funciton parameters or declared inside a funciton can be referenced only in that function, and are known as local.

Nested Scope

JavaScript distinguishes not just global and local bindings. Blocks and functions can be created inside other blocks and functions, producing multiple degrees of locality.

The set of bindings visible inside a block is determined by the place of that block in the program test. Each local scope can also see all the local scopes that contain it, and all scopes can see the global scope. This approach to binding visibility is called lexical scoping.

Funtions as Values

A functino binding usually just acts as a name for a specific piece of the program. Such a binding is defined once and never changed. This makes it easy to confuse a function and its name. But the two are different. A function value can do all the things That other values do -- You can use it in arbitrary expressions, not just call it. It is possible to store a function value in a new binding, pass it as an argument to a function, and so on.

Declaration Notation

This is a function declaration. The statement defines the binding square and points it at the given function. It is slightly easier to write and doesnt require a semicolon after the function. 

 
                  function square(x) {
                  return x * x; 
                }

Arrow Funcitons

Instead of the function keyword, it uses an arrow (=>) 


                  const pwer = (base, exponent) => {}

It expresses something like, "this input (the parameters) produces this result (the body). When there is only one parameter name, you can omit the parentheses around the parameter list. If the body is a single expression, rather than a block in braces, that expression will be returned from the function.

The Call Stack

The place where the computer stores the context of a program is known as the call stack. Every time a function is called, the current context is stored on top of this stack. When a function returns, it removes the top context from the stack and uses that context to continue execution.

Storing this stack requires space in the computer's memory. When the stack grows too big, the computer will fail with a message like, "out of stack space."

Optional Arguments

JavaScript is extremely broad-minded about the number of arguments you pass to a function. If you pass too many, the extra ones are ignored. If you pass too few, the missing parameters get assigned the value undefined.

The downside of this is that it is possible -likely, even- that you'll accidenttally pass the wrong number of arguments to functions. And no one will tell you about it. The upside is that this behavior can be used to allow a function to be called with different numbers of arguments.

Closure

Being able to reference a specific instance of a local binding in an enclosing scope is called closure. A function that references bindings from local scopes around it is called a closure. This behavior not only frees you from having to worry about lifetimes of bindings but also makes it possible to use function values in some creative ways.

A good mental model is to think of function values as containing both the code in their body and the environment in which they are created. When called, the function body sees the environment in which it was created, not the environment in which it is called.

Recursion

It is perfectly okay for a funciton to call itself, as long as it doesn't do it so often that it overflows the stack. A function that calls itself is called recursive. This is a more complicated topic that requires some work to fully grasp. There is better resources than this section in my notes out there, but here are some small thoughts I liked in the book.

  • Running a simple loop is generally cheaper than calling a function multiple times.
  • Worrying about efficiency can be a distraciton.
  • Recursion most often solves problems that require exploring or processing several "branches".

Growing Functions

There are two more or less natural ways for funcitons to be introduced into programs. The first is that you find yourself writing similar code muliple times. You'd prefer not to do that.

"KEEP YOUR CODE DRY"

Don't Repeat Yourself.

The second way is that you find you need some functionality that you haven't written yet and that sounds like it deserves its own function. Instead of lifting out the repeated part of our program wholesale, let's try to pick out a single concept. A useful principle is to not add cleverness unless you are absolutely sure you're going to need it.

Funcitons and Side Effects

Functions can be roughly divided into those that are called for their side effects and those that are called for their return value. A pure function is a specific kind of value-producing function that not only has no side effects but also doesn't rely on side effects from other code. A pure function has the pleasant property that, when called with the same arguments, it always produces the same value (and doesn't do anything else). A clal to such a function can be substitued by its return valeu without changing the meaning of the code.

Summary

The function keyword, when used as an expression, can create a function value. When used as a statement, it can be used to declare a binding and give it a function as its value. Arrow functions are yet another way to create functions.

A key aspect in understanding functions is understanding scoopes. Each block creates a new scope. Parameters and bindings declared in a given scope are local and not visible from the outside.

Separating the tasks your program performs into different funcitons is helpful. You won't have to repeat yourself as much, and functions can help organize a program by grouping code into pieces that do specific things.

Chapter 4: Data Structures: Objects and Arrays.

Notes for Chapter 4.

Properties

Almost all JavaScript values have properties. The exceptions are null and undefiened. The two ways to access properties in JavaScript are with a dot and with square brackets : value.x and value[x]. The difference is how the x is intepreted. When using a dot, the word after the dot is the literal name of the property. When using square brackets, hte expression between the brackets is evaluated to get the property name.

Methods

Properties that contain functions are generally called methods of the value they belong to, as in "toUpperCase is a method of a string.""

Objects

The binary in operator, when applied to a string and an object, tells you wheter that object has a property with that name. The difference between setting a property to undefined and deleting it is that, in the first case, the object still has the property. There's an Object.assign function that copies all properties from one object into another.

Mutability

With objects, there is a difference between having two referrences to the same object and having two different objects that contain the same properties. When you compare objects with JavaScript's == operator, it compares by identity: it will produce true only if both objects are precisely the same value. Comparing different objects will return false, even if they have identical properties. There is no 'deep' comparison operation built in. Most require recursive functions to access the properties.
